# LC 0877 - Stone Game

## Problem

Alex and Lee play a game with piles of stones arranged in a row (even number of piles, each with positive stones). Players take turns picking an entire pile from either end. Alex goes first. The player with the most stones wins. Return `true` if Alex wins, assuming both play optimally.

## Approach

### Mathematical Property

When the number of piles is even, the first player **always wins**:

1. Piles are indexed 0, 1, 2, ..., n-1 (n is even)
2. First player can choose pile 0 or pile n-1
3. This forces the opponent to always pick from piles of one parity (odd or even indices)
4. First player calculates which parity sum is larger and takes that group

Therefore: `return True` for O(1) solution.

### DP Approach (General Solution)

Same as LC 486 - Predict the Winner:

```
dp[i][j] = max(
    piles[i] - dp[i+1][j],   # pick left
    piles[j] - dp[i][j-1]    # pick right
)
```

Return `dp[0][n-1] > 0`

## Complexity

- **Optimal**: O(1) time, O(1) space (mathematical property)
- **DP**: O(n^2) time, O(n^2) space

## Key Points

1. **Even-length property**: first player can always control which parity group to take
2. **DP generality**: the DP approach works for any number of piles, not just even
3. **Game symmetry**: both players play optimally
