# LC 0600 - Non-negative Integers without Consecutive Ones

## Problem

Given a positive integer `n`, find the number of non-negative integers less than or equal to `n` whose binary representations do NOT contain consecutive ones.

## Examples

```
Input: n = 5
Output: 5
Explanation:
0: 0, 1: 1, 2: 10, 3: 11 (consecutive 1s), 4: 100, 5: 101
Valid: 0, 1, 2, 4, 5 -> 5
```

## Approach

### Digit DP on Binary Representation

**Key techniques**:
1. Process binary digits instead of decimal digits
2. State only needs to track whether the previous bit was 1
3. Skip when `prev == 1 and digit == 1` (consecutive ones)

### State Definition

```
dp[pos][prev][limit]
- pos: current binary bit position (high to low)
- prev: value of the previous bit (0 or 1)
- limit: whether constrained by n's binary representation
```

## Solution

```python
from functools import lru_cache

def findIntegers(n: int) -> int:
    s = bin(n)[2:]
    length = len(s)

    @lru_cache(None)
    def dfs(pos, prev, limit):
        if pos == length:
            return 1  # found a valid number

        total = 0
        max_digit = int(s[pos]) if limit else 1

        for digit in range(max_digit + 1):
            if prev == 1 and digit == 1:
                continue  # consecutive ones
            total += dfs(pos + 1, digit, limit and digit == max_digit)

        return total

    return dfs(0, 0, True)
```

## Complexity

- **Time**: O(log n) - number of binary digits
- **Space**: O(log n) - recursion depth

## Key Points

1. **Digit DP framework**: standard template for counting numbers with constraints in a range
2. **State design**: `prev` tracks the previous bit to detect consecutive ones
3. **Limit handling**: ensures count stays within [0, n]
4. **Memoization**: avoids recomputing identical states
