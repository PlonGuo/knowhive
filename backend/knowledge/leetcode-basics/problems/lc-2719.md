# LC 2719 - Count of Integers

## Problem

Given two numeric strings `num1` and `num2`, and two integers `min_sum` and `max_sum`, return the count of integers in the range `[num1, num2]` whose digit sum is in `[min_sum, max_sum]`. Return the answer modulo 10^9 + 7.

## Approach

### Digit DP

Standard digit DP with range decomposition: `f(num2) - f(num1 - 1)`.

### State Design

- `pos`: current digit position
- `sum`: current digit sum
- `tight`: whether constrained by the upper bound
- `lead_zero`: whether there are leading zeros

### Memoized Search

```python
MOD = 10**9 + 7

def dfs(pos, digit_sum, tight, lead_zero, num_str):
    if pos == len(num_str):
        return 1 if min_sum <= digit_sum <= max_sum else 0

    if not tight and not lead_zero and memo[pos][digit_sum] != -1:
        return memo[pos][digit_sum]

    limit = int(num_str[pos]) if tight else 9
    total = 0

    for digit in range(0, limit + 1):
        next_tight = tight and (digit == limit)
        next_lead_zero = lead_zero and (digit == 0)
        next_sum = digit_sum + digit

        # Pruning: if sum already exceeds max_sum, skip
        if next_sum > max_sum:
            continue

        total += dfs(pos + 1, next_sum, next_tight, next_lead_zero, num_str)
        total %= MOD

    if not tight and not lead_zero:
        memo[pos][digit_sum] = total

    return total
```

## Complexity

- **Time**: O(n * sum * 10) where n = digit count, sum = max digit sum
- **Space**: O(n * sum) for memoization

## Key Points

1. **Range decomposition**: use `f(high) - f(low - 1)` technique
2. **Leading zero handling**: leading zeros don't affect digit sum but need special treatment
3. **Pruning**: terminate early when digit sum exceeds `max_sum`
4. **Modular arithmetic**: apply mod during addition to prevent overflow
