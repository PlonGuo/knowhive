# LC 0732 - My Calendar III

## Problem

Implement a `MyCalendarThree` class to store events. You can always add a new event. For each `book(start, end)` call (half-open interval `[start, end)`), return the maximum K-booking — the largest K such that K events have some non-empty intersection.

## Approach

### Difference Array Method

**Key observation**: each booking increments the count for all time points in `[start, end)` by 1.

**Algorithm**:
1. Use a dictionary to store change points
2. For each booking `[start, end)`:
   - `diff[start] += 1` (count increases from start)
   - `diff[end] -= 1` (count decreases from end)
3. Scan all time points in order, accumulate changes, track the maximum

### Why Difference Array?

- **Efficient update**: range update modifies only two endpoints, O(1)
- **Lazy evaluation**: actual counts computed only at query time
- **Space efficient**: stores only change points, not all time points

## Solution

### Using dict + sorting

```python
class MyCalendarThree:
    def __init__(self):
        self.diff = {}

    def book(self, start: int, end: int) -> int:
        self.diff[start] = self.diff.get(start, 0) + 1
        self.diff[end] = self.diff.get(end, 0) - 1

        sorted_times = sorted(self.diff.keys())

        max_k = 0
        curr = 0

        for time in sorted_times:
            curr += self.diff[time]
            max_k = max(max_k, curr)

        return max_k
```

### Walkthrough

```
book(10, 20) -> diff: {10:1, 20:-1}
Scan: t=10 curr=1 max=1, t=20 curr=0 -> return 1

book(50, 60) -> diff: {10:1, 20:-1, 50:1, 60:-1}
Scan: max=1 -> return 1

book(10, 40) -> diff: {10:2, 20:-1, 40:-1, 50:1, 60:-1}
Scan: t=10 curr=2 max=2, t=20 curr=1, ... -> return 2
```

## Complexity

### dict + sorting

- **book**: O(n log n) per call (sorting)
- **Space**: O(n) - change points

### SortedDict optimization

- **book**: O(log n + n) - O(log n) insert, O(n) scan

## Key Points

1. **Difference array idea**: range update becomes endpoint modification
2. **Half-open interval**: `[start, end)` includes start, excludes end
3. **Scan order**: must scan time points in sorted order
4. **Maximum overlap**: the running maximum during scan is the K-booking answer
