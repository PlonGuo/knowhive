# LC 0486 - Predict the Winner

## Problem

Given an integer array `nums`. Two players take turns picking numbers from either end of the array (one number per turn). Player 1 goes first. Return `true` if Player 1 can win (or tie), assuming both play optimally.

## Approach

### Core Idea

- Zero-sum game solved with interval DP
- `dp[i][j]` = maximum score advantage the current player can achieve over the opponent in interval `[i, j]`
- Current player picks either `nums[i]` or `nums[j]`, then the opponent plays optimally on the remaining interval

### State Transition

```
dp[i][j] = max(
    nums[i] - dp[i+1][j],   # pick left end
    nums[j] - dp[i][j-1]    # pick right end
)
```

### Base Case

- When `i == j`: `dp[i][j] = nums[i]` (only one number left)
- Final answer: `dp[0][n-1] >= 0` (Player 1's advantage is non-negative)

## Complexity

- **Time**: O(n^2) - fill n*n DP table
- **Space**: O(n^2) - DP table

## Key Points

1. **Game theory perspective**: current player's score minus opponent's optimal score on the remaining interval
2. **Interval DP order**: compute by increasing interval length
3. **Optimal strategy**: both players play optimally, making the problem symmetric
