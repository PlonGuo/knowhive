# LC 0834 - Sum of Distances in Tree

## Problem

Given an undirected connected tree with `n` nodes (labeled 0 to n-1) and `n-1` edges, return an array `answer` where `answer[i]` is the sum of distances between node `i` and all other nodes.

## Approach

### Rerooting DP (Two-Pass DFS)

Classic rerooting DP problem. Brute force O(n^2) would TLE; rerooting DP achieves O(n).

### Pass 1: Compute Subtree Info

Choose any root (e.g., 0). For each node u, compute:
- `subtree_size[u]`: number of nodes in u's subtree
- `dp[u]`: sum of distances from u to all nodes in u's subtree

Transition:
```
subtree_size[u] = 1 + sum(subtree_size[children])
dp[u] = sum(dp[child] + subtree_size[child])
```

### Pass 2: Rerooting

Starting from `ans[0] = dp[0]`, propagate to children:
```
ans[v] = ans[u] - subtree_size[v] + (n - subtree_size[v])
```

**Intuition**: when switching root from u to v:
- Nodes in v's subtree (`subtree_size[v]` nodes): distance decreases by 1
- All other nodes (`n - subtree_size[v]` nodes): distance increases by 1

## Complexity

- **Time**: O(n) - two DFS passes
- **Space**: O(n) - subtree sizes, DP results, adjacency list

## Key Points

1. **Rerooting formula**: understand the mathematical relationship of distance changes
2. **Subtree size**: must know node count in each subtree
3. **Undirected graph**: avoid revisiting parent nodes during DFS
4. **Initialization**: choose any node as root for the first pass
