# LC 0253 - Meeting Rooms II

## Problem

Given an array of meeting time intervals `intervals` where `intervals[i] = [start_i, end_i]`, return the minimum number of conference rooms required.

## Approach

### Greedy + Min-Heap

**Key observation**: we need to track the earliest-ending meeting among all currently active meetings.

**Algorithm**:
1. Sort all meetings by start time
2. Use a min-heap to store end times of active meetings
3. For each meeting:
   - If the heap top (earliest ending) <= current meeting's start time: room is free, pop and reuse
   - Otherwise: need a new room
4. Heap size = minimum rooms needed

### Why Min-Heap?

- O(log n) access to the earliest ending meeting
- If the earliest ending meeting hasn't finished, no other meeting has either
- If it has finished, its room can be reused

## Solution

```python
import heapq
from typing import List

def minMeetingRooms(intervals: List[List[int]]) -> int:
    if not intervals:
        return 0

    intervals.sort(key=lambda x: x[0])

    heap = []  # min-heap of end times

    for interval in intervals:
        start, end = interval

        if heap and heap[0] <= start:
            heapq.heappop(heap)  # reuse room

        heapq.heappush(heap, end)

    return len(heap)
```

### Walkthrough

```
Input: intervals = [[0,30],[5,10],[15,20]]

Sorted: [[0,30],[5,10],[15,20]]

1. [0,30] -> heap = [30] (1 room)
2. [5,10] -> heap[0]=30 > 5, need new room
           -> heap = [10, 30] (2 rooms)
3. [15,20] -> heap[0]=10 <= 15, reuse room
           -> pop 10, push 20 -> heap = [20, 30] (2 rooms)

Result: 2
```

## Complexity

- **Time**: O(n log n) - sorting + heap operations
- **Space**: O(n) - worst case all meetings overlap

## Key Points

1. **Sorting is key**: must sort by start time to process chronologically
2. **Min-heap role**: tracks the earliest ending active meeting
3. **Reuse condition**: `heap[0] <= start` means a room is available
4. **Heap size**: always represents the count of currently active meetings
5. **Edge cases**: empty array (return 0), single meeting (return 1), all meetings overlap (return n)
